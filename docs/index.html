Basics

API url:
https://lpw-public-api.herokuapp.com/

Definitions and references

Definitions:

  1. property object - JSON object with real estate property data
  2. localized - translated (if translation is present) accordingly to locale set in constructor or passed as request parameter
  3. price on demand,
    deposit on demand,
    commission on demand - on_demand=true means that price (or deposit, or commission) isn't public. I.e. it can be provided only by a seller/owner privately. (Contact form request, phone call, etc.)
  4. slug - part of Semantic URL

References:

  1. Country code format: ISO 3166-1
  2. Currency code format: ISO 4217
  3. Locales codes format: ISO 639-1
  4. Timestamps format: https://www.w3.org/TR/NOTE-datetime

Authentication

Use simple token-based authorization.
Send it in the header of your request:
Authorization: Token token=YOUR_TOKEN_HERE

Token may be obtained from your profile settings in mylpw.io

Filters: Location

Location point
Geographical point. Allows to search for a properties around this point accordingly to specified radius.
Name Type Examples Description
location_point object JSON:
{"location_point": {
    "lat": 50.0755381,
    "lon": 14.43780049999998,
    "radius": 5,
    "country_code": "CZ"
}}
Query string:
?location_point[country_code]=CZ &location_point[lat]=50.0755381 &location_point[lon]=14.43780049999998 &location_point[radius]=5
  • lat - latitude
  • lon - longitude
  • radius (km) - radius from the point to search in
  • country_code - allows to show properties from this country at first place, because some geographical area can contain properties from different countries
Location shape
Geographical rectangle consisting of 2 geographical points (top-right and bottom-left). Allows to search for properties inside of this rectangle.
Name Type Examples Description
location_point object JSON:
{"location_shape": {
    "top_right": {
        "lat": 50.177403,
        "lon": 14.7067945
    },
    "bottom_left": {
        "lat": 49.94193629999999
        "lon": 14.22445330000005
    },
    "country_code": "CZ"
}}
Query string:
?location_shape[bottom_left][lat]=49.94193629999999 &location_shape[bottom_left][lon]=14.22445330000005 &location_shape[country_code]=CZ &location_shape[top_right][lat]=50.177403 &location_shape[top_right][lon]=14.7067945
  • top_right - geographical point
  • bottom_left - geographical point
  • country_code - allows to show properties from this country at first place, because some geographical area can contain properties from different countries

Filters: Price

Allows to filter properties by price
Name Type Examples Description
price object JSON:
{
    "price": {
        "min": 100000,
        "max": 150000,
        "currency": 1,
        "period": "week"
    }
}
Query string:
?price[currency]=1&price[max]=150000 &price[min]=100000&price[period]=week 		Range value, so only min, or only max, or both allowed.
  • min - minimum price
  • max - maximum price
  • currency - currency ID. See currencies query.
  • period - for_rent=true only
    Allowed values: 'day' | 'week' | 'month'
