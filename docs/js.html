Documentation

Definitions and references

Definitions:

  1. localized - translated (if translation is present) accordingly to locale set in constructor or passed as request parameter
  2. price on demand,
    deposit on demand,
    commission on demand - on_demand=true means that price (or deposit, or commission) isn't public. I.e. it can be provided only by a seller/owner privately. (Contact form request, phone call, etc.)
  3. slug - part of Semantic URL

References:

  1. Country code format: ISO 3166-1
  2. Currency code format: ISO 4217
  3. Locales codes format: ISO 639-1
  4. Timestamps format: https://www.w3.org/TR/NOTE-datetime

Initialization

var lpw = new LPW(token, options);

Parameters

Name Type Default Description
Token String - LPW front-end user token for API access.
options Object - Options for LPW instance: 
{
    locale      : {String},
    debugEnabled: {Boolean}
}
Where:
  • locale - allows to set default locale for all requests.
    List of available locales depends on your settings in mylwp.io.
    Full list of available locales can be found here.
  • debugEnabled - enables/disables debug info

Available methods


getProperties

lpw.getProperties(options,callback);

Parameters

Name Type Default Description
options Object - Set of options to filter properties. See full description below
callback Function - Function called when request is complete. See full description below

Options

List of available properties:

Name Type Default Description
forSale Boolean true Show properties only for sale. Should has opposite value for forRent
forRent Boolean false Show properties only for rent. Should has opposite value for forSale
locale String 'en' Language of property data (title, description, etc.).
List of available locales depends on your settings in mylwp.io.
Full list of available locales can be found here.
locationShape Object - Geographical shape consisting of 2 geographical points (top-right and bottom-left).
Example: 
{
   top_right   : {
     lat: {Number},
     lon: {Number}
   },
   bottom_left : {
     lat: {Number}
     lon: {Number}
   },
   country_code: {String}
}
Country code to show properties from this country first, because shape can contain properties from different countries.
locationPoint Object - Geographical point.
Example: 
{
    lat         : {Number},
    lon         : {Number},
    radius      : {Number},
    country_code: {String}
}
Where:
  • Radius: radius from the point (km) to search in.
  • Country code: allows to show properties from this country at first place, because some geographical area can contain properties from different countries.
price Object -

Property price filter. It should contain currency code and at least one of min or max value.
Additionally, for rent query, it should contain rent period.

Example: 
{
    min: {Number},
    max: {Number},
    currency: {Number},
    period: {String}
}
Where:
  • min - minimum price value
  • max - maximum price value
  • currency - currency id. Can be obtained by getCurrencies method.
  • period - Rent period. Used only in queries with forRent=true.
    Allowed values: 'day' | 'week' | 'month'
area Object - Property area filter (sq. m).
Example: 
{
    min: {Number},
    max: {Number}
}
propertyTypes Array - Property type filter.
Allowed values: 1 - for apartments, 2 - for houses.
You can use one or both, example: [2] or [1,2]
rooms Array - Rooms number filter.
Allowed values: 1, 2, 3, 4, 5.
Where 5 means 5 and above.
You can use one or few, example: [5] or [1,4,5]
orderBy String 'desc' Order filter.
Allowed values: 'asc' and 'desc'.
hdPhotos Boolean false Show properties only with high-quality photos.
page Number 1 Page number.
perPage Number 10 Properties per page.
id Number - Property ID. Other filters except forSale, forRent, locale are ignored.
ids Array - Array of properties IDs. Other filters except forSale, forRent, locale, orderBy are ignored.
longRent Boolean - Rent only!
Show properties only for long-term rent. Can be used with shortRent, but in case when both values set to true would give same result, as both set to false or undefined.
shortRent Boolean - Rent only!
Show properties only for short-term rent. Can be used with longRent, but in case when both values set to true would give same result, as both set to false or undefined.
childFriendly Boolean - Rent only!
Show only child-friendly properties for rent.
petsAllowed Boolean - Rent only!
Show properties only where pets allowed.
persons Number - Rent only!
Show properties only where specified number of persons allowed.

Callback

It will receive answer as first argument. Answer is an object with these properties:

{
    data: {
        property_objects: [
            {Object},
            {Object}
            ...
        ],
        total: {Number},
        offmarket: {Number}
    },
    status: {Number}
    statusText: {String}
}
Name Type Description
data Object Payload object, where:
  • property_objects - array with properties data.
  • total - total number of properties matching request.
  • offmarket - total number of off-market properties matching request.
status Number XMLHttpRequest.status
statusText String XMLHttpRequest.statusText

getPDF

lpw.getPDF(propertyId, options, callback);

Parameters

Name Type Default Description
propertyId Number - Property ID
options Object -
  • forRent {Boolean} - Get PDF with information about property for rent (if available). Information for sale is default.
  • locale {String} - PDF locale. If not specified - locale of constructor will be used.
callback Function - Function called when request is complete. Description is below

Callback

It will receive answer as first argument. Answer is an object with these properties:

{
    data: {
        pdf_path: {String}
    },
    status: {Number}
    statusText: {String}
}
Name Type Description
data Object | Null If answer received successfully:
  • pdf_path {String} - URL to PDF file
If fails - data will be null
status Number XMLHttpRequest.status
statusText String XMLHttpRequest.statusText

  • rent_short.sort_price - Object (see description below)
    or null if not present.
